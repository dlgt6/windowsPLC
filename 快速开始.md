# 快速开始指南

## 前置要求

在开始之前，请确保已安装以下软件：

1. **.NET 8.0 SDK**
   - 下载地址：https://dotnet.microsoft.com/download/dotnet/8.0
   - 选择 "SDK" 版本进行安装
   - 安装后，在命令行输入 `dotnet --version` 验证安装

2. **Git**（可选，用于版本控制）
   - 下载地址：https://git-scm.com/downloads

## 本地开发步骤

### 方法一：使用命令行

1. **打开命令提示符或PowerShell**

2. **进入项目目录**
   ```bash
   cd C:\Users\Administrator\Desktop\自动化计算工具-5
   ```

3. **恢复依赖**
   ```bash
   dotnet restore
   ```

4. **构建项目**
   ```bash
   dotnet build --configuration Release
   ```

5. **运行程序**
   ```bash
   dotnet run
   ```

### 方法二：使用构建脚本（推荐）

直接双击运行 `build.bat` 文件，脚本会自动完成以下步骤：
- 清理之前的构建
- 恢复依赖
- 构建项目
- 发布为单文件exe
- 启动程序

### 方法三：使用Visual Studio

1. **打开项目**
   - 双击 `自动化计算工具.csproj` 文件
   - 或在Visual Studio中选择 "打开项目或解决方案"

2. **生成解决方案**
   - 按 `F6` 或点击菜单 "生成" → "生成解决方案"

3. **运行程序**
   - 按 `F5` 或点击 "开始调试" 按钮

## 发布为独立exe文件

### 使用命令行发布

```bash
dotnet publish -c Release -r win-x64 --self-contained true -p:PublishSingleFile=true -p:IncludeNativeLibrariesForSelfExtract=true -p:EnableCompressionInSingleFile=true -p:PublishTrimmed=true -p:TrimMode=link -p:PublishReadyToRun=true -o ./publish
```

发布后的文件位于 `./publish` 目录，主要文件为 `自动化计算工具.exe`。

### 使用Visual Studio发布

1. 右键项目 → 发布
2. 选择目标：文件夹
3. 点击 "下一步"
4. 配置发布设置：
   - 目标框架：net8.0-windows
   - 目标运行时：win-x64
   - 部署模式：自包含
   - 生成单个文件：是
   - 裁剪程序集：是
   - 准备运行时：是
5. 点击 "发布"

## 常见问题

### Q: 提示找不到dotnet命令？
A: 请确保已安装.NET 8.0 SDK，并将其添加到系统PATH环境变量中。

### Q: 编译时出现错误？
A: 请检查.NET SDK版本是否为8.0或更高版本。

### Q: exe文件太大？
A: 项目已配置了多种优化选项（单文件发布、裁剪、压缩），生成的exe文件应该只有5-8MB。如果仍然太大，可以尝试移除不必要的依赖。

### Q: 程序无法启动？
A: 请确保：
   - 操作系统为Windows 10/11
   - 已安装最新的.NET运行时（如果使用依赖发布）

## 上传到GitHub

1. **初始化Git仓库**（如果还没有）
   ```bash
   git init
   ```

2. **添加所有文件**
   ```bash
   git add .
   ```

3. **提交更改**
   ```bash
   git commit -m "Initial commit"
   ```

4. **连接远程仓库**
   ```bash
   git remote add origin <your-repository-url>
   ```

5. **推送到GitHub**
   ```bash
   git branch -M main
   git push -u origin main
   ```

6. **创建发布版本**
   ```bash
   git tag v1.0.0
   git push origin v1.0.0
   ```

GitHub Actions会自动构建并创建Release。
